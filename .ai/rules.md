# Project Rules

1. Read code and deploy scripts before changing behavior.
2. Keep changes scoped to the requested task.
3. Do not delete `F:\newAPI`, `/data/new-api/**`, `new-api`, or `new-api-staging`.
4. Do not store secrets in Git or `.ai/`.
5. Production Caddy changes require explicit user confirmation.
6. Staging deploys may proceed when the user clearly asks for staging.
7. Keep migration reports factual: counts, paths, warnings, and validation results only.
8. Preserve sub2api LGPLv3 attribution and source references.
9. Old new-api AI docs are legacy context, not active coding rules.
