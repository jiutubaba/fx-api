## 2026-05-18 - Account and Channel Visibility Follow-up

### Scope

- Improve the admin account management page after production cutover.
- Investigate why user-facing available channels and channel status were blank in production.
- Keep old new-api resources preserved while deploying the fix to the new Fuxi API runtime.

### Findings

- Production feature flags were enabled:
  - `available_channels_enabled = true`
  - `channel_monitor_enabled = true`
- Production data shape after migration:
  - `channels`: 0 rows
  - `accounts`: 101 rows, including 29 active accounts
  - `groups`: 5 rows, all `openai` and active
  - `channel_monitors`: 0 rows
- Available channels were blank because the new sub2api UI/API read the formal `channels` table, while migrated usable resources live in `accounts` and `account_groups`.
- Channel status was blank because no monitor tasks/history had been configured or migrated. This was not a disabled setting.

### Code Changes

- Commit: `409cb4ee Improve account and channel visibility`
- `frontend/src/views/admin/AccountsView.vue`
  - Added an old-repo-style statistics strip for total accounts, enabled accounts on the current page, schedulable accounts on the current page, and abnormal/disabled accounts on the current page.
- `backend/internal/handler/available_channel_handler.go`
  - Added an account-pool fallback for `/api/v1/channels/available` when `channels` is empty and the user has accessible groups.
  - The fallback does not fabricate supported models when the production group has no default model or route configuration.
- `backend/internal/handler/available_channel_handler_test.go`
  - Added unit coverage for the fallback behavior.
- `frontend/src/views/user/ChannelStatusView.vue`
  - Added read-only account-pool status cards when real monitor data is empty.
  - Synthetic fallback cards do not open the monitor detail dialog.
- `frontend/src/api/channelMonitor.ts`
  - Allowed empty monitor status/availability values and marked synthetic rows.
- `frontend/src/components/user/monitor/MonitorHero.vue`
  - Added an `unmonitored` overall state so missing monitor configuration is not shown as degraded.
- `frontend/src/i18n/locales/zh.ts` and `frontend/src/i18n/locales/en.ts`
  - Added translations for account pool stats and unmonitored status.
